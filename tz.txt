В текстовом файле хранятся пары вида <Имя>=<Значение>, где <Имя> – это идентификатор (последовательность латинских букв и цифр, начинающаяся с буквы, дополнительно может содержать символ подчёркивания «_», может начинаться с символа подчёркивания); <Значение> – некоторая строка. Допускается наличие пар, которые имеют одинаковое имя или значение. В <Имени> и <Значении> регистр букв не различается. В файле могут присутствовать пустые строки.
Пример текстового файла:
...
count = 100
Property1 = 10
property1 = 1,5
Path = C:\Windows
cpu=Intel Core2 Duo

ip_address = 127.0.0.1
Comment = Файл с исходными данными
...

Необходимо разработать консольное java приложение для обработки данных, хранящихся в текстовом файле. Графический интерфейс (GUI), аналогичный окнам в Windows, в приложении разрабатывать не требуется.
В программе должно быть реализовано простейшее меню, например, 1 – загрузить из файла, 2 – добавить пару, 3 – удалить пару, … , 0 – выход, и т.д.
Пользователь должен иметь возможность выполнить следующие действия:
Основное задание (обязательное для выполнения)
1. Передать файл с исходными данными через командную строку при старте программы.
2. Стартовать программу без передачи файла через командную строку.
3. Загрузить пары из произвольного текстового файла во время работы программы.
Программа должна считывать из файла только те пары, в которых <Имя> является идентификатором (см. выше). Пары, в которых <Имя> не является идентификатором, игнорируются, и соответствующее сообщение должно выводиться на консоль.
4. Сохранить список пар в текстовом файле во время работы программы.
5. Посмотреть текущий список всех пар на экране.
6. Добавить новую пару в список. 
Пользователь вводит новую пару в виде <Имя> = <Значение>. Программа должна самостоятельно выделить из строки <Имя> и <Значение> пары.
7. Удалить существующую пару из списка. Пользователь вводит имя пары, которую требуется удалить. Из списка пар должны быть удалены все пары с заданным именем.
8. Сортировать пары по имени и по значению, по возрастанию и убыванию.
9. Реализовать фильтр пар по имени и по значению.
Фильтр по имени: пользователь вводит <Имя> пары, после чего на экране отображаются только те пары, у которых <Имя> совпадает с заданным (регистр букв не учитывается).
Фильтр по значению: пользователь вводит <Значение> пары, после чего на экране отображаются только те пары, у которых <Значение> совпадает с заданным (регистр букв не учитывается).

Дополнительные требования
Программа должна корректно обрабатывать наиболее вероятные ситуации, которые могут возникнуть при работе с ней. Например, пользователь должен получить соответствующее сообщение:
при попытке считать данные из несуществующего файла;
при попытке добавить новую пару <Имя> = <Значение>, в которой <Имя> не является идентификатором, и т.д.

Разрабатывать систему помощи для приложения не требуется.
